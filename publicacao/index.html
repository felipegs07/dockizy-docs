Publicação

Depois que sua documentação estiver pronta, é hora de publicar. Para isso existe alguns comandos e várias formas de gerar o site para publicação.

Gerar Arquivos Estáticos

A forma mais simples, gera todo seu site em arquivos estáticos feitos em HTML e CSS numa pasta chamada public. Com essa pasta, é possível publicar seu site de várias formas, como FTP e etc.

Para fazer isso é necessário o comando:

    hexo generate

Deploy Automático

Para fazer o deploy automático, é necessário modificar algumas configurações no _config.yml.
Uma configuração de implantação válida deve ter um type que indicará o serviço que será utilizado. Por exemplo:

    deploy:
        type: git

Implantação com Git

Para utilizar o GitHub Pages ou o git para implantação é necessário instalar um pacote pelo npm

    npm install hexo-deployer-git --save

Depois é necessário editar as configurações no arquivo _config.yml

    deploy:
        type: git
        repo: <repository url>
        branch: [branch]
        message: [message]
  • repo: URL do repositório no GitHub/Bitbucket/Coding/GitLab.
  • branch: Qual dos branchs o deploy será implementado. Ele identifica automaticamente o GitHub.
  • message: A mensagem customizada que será enviada no commit. O padrão é (Site updated: [data])

Por exemplo:

    deploy:
        type: git
        repo: https://github.com/felipegs07/dockizy.git

Nesse exemplo ocultando o message para que seja usado a mensagem padrão.

Depois de configurar é isso, é possível fazer o deploy automático. Para isso use o comando:

    hexo deploy

E pronto, sua documentação estará publicada e pronto para ser utilizada.
